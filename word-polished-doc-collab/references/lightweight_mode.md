# Lightweight Mode

**这份文档的定位。** 本文是 `word-polished-doc-collab` 的轻量模式入口，只保留你最常见、最刚性的格式要求，以及一个足够简单的 workspace 约定。它服务“先把文档转顺、转美观、快速可交付”的任务，不默认引入质量 gate 和重型基础设施。

## 什么时候走轻量模式

**轻量模式适合快任务。** 典型场景包括：
- 单篇或少量文档快速转 `docx`
- 规则已经明确，重点是把版式落对
- 当前不需要 Office 原生 chart、复杂模板继承、OOXML patch、自动 QA

## 最小 workspace

**轻量模式的 workspace 可以很简单。** 推荐结构：

```text
doc-job/
├── doc.md
├── assets/
└── out/
```

**如果任务是从原始 Word 起步，可以再加一个输入文件。** 推荐结构：

```text
doc-job/
├── input.docx
├── doc.md
├── assets/
└── out/
```

**轻量模式不强制 `meta.json`、不强制 `temp/`、不强制验证目录。** 先把 Markdown 和最终 `.docx` 跑顺，再决定要不要升级到精细模式。

## 你的默认格式要求

**默认字体 profile 是 `中文宋体 + 英文 Times New Roman`。** 正文、普通标题、表题、图题、表注默认都从这套 profile 出发。

**正文默认使用小四。** 具体要求是：
- 中文字体：`宋体`
- 英文字体：`Times New Roman`
- 字号：`小四 12pt`
- 行距：`1.5`
- 段前：`0.5` 行
- 段后：`0.5` 行

**标题按重要性逐级放大。** 轻量模式推荐梯度：
- 文档主标题：`22pt`
- 一级标题：`16pt`
- 二级标题：`15pt`
- 三级标题：`14pt`
- 四级标题：`12pt`

**标题和正文共享段落节奏。** 标题默认也是：
- 行距：`1.5`
- 段前：`0.5` 行
- 段后：`0.5` 行

**表格使用独立密度。** 具体要求是：
- 默认表格字号：`五号 10.5pt`
- 密表回退字号：`小五 9pt`
- 段前：`0`
- 段后：`0`

**表题在表上方。** 具体要求是：
- 位置：表格前一段
- 字号：默认 `12pt`
- 段前：`0.5` 行
- 段后：`0`

**图题在图下方。** 具体要求是：
- 位置：图片后一段
- 字号：默认 `10.5pt`
- 段前：`0`
- 段后：`0.5` 行

**表注在表下方。** 具体要求是：
- 位置：表格后一段
- 字号：默认 `10.5pt`
- 段前：`0`
- 段后：`0.5` 行

## 可选字体组合

**正式强调可以切到 `中文楷体 + 英文 Times New Roman`。** 这套组合适合引用、签批说明、法规摘录和局部强调，不建议把整篇连续正文都切成楷体。

**展示型标题可以切到 `中文黑体 + 英文 Arial`。** 这套组合适合封面、章节标题、图表短标题和标签，不建议把整篇正式正文都切成黑体。

## 轻量模式的执行原则

**先把语义写清楚，再做版式。** Markdown 里至少要分清标题、正文、表格、图片、表题、图题和表注。

**先走最简单的构建路线。** 默认优先 `markdown -> docx` 或 `docx -> markdown -> docx`，不要一开始就引入复杂 OOXML patch。

**轻量模式先不做重质检。** 不默认引入质量 gate、字体槽位检查或单独验证目录，除非用户主动要求，或者任务已经明显升级成精细模式。
